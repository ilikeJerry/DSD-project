# DSD — Decision Support Dashboard (Clinical Operations)

## 한 줄 요약

중환·감시 구역에서 **누가 위험한지**, **데이터를 얼마나 믿을 수 있는지**, **연결이 흔들릴 때도 조용히 실패하지 않는지**를 한 화면에서 잡아 주는 **운영 콘솔 UI**입니다.  
법적 SaMD 완결 세트가 아니라, **실제 병동 운영에 가까운 판단·교대·복구 경험**을 보여 주기 위한 제품형 스켈레톤입니다.

---

## 이 제품이 푸는 문제

| 운영 현장의 문제 | DSD가 보여 주는 방향 |
|------------------|----------------------|
| 알림만 많고 우선순위가 안 보인다 | **우선순위 큐**로 “지금 누구를 볼지”가 한눈에 정렬됩니다. |
| 연결 끊겼을 때 화면만 멀쩡해 보인다 | **재연결·제한 모드**를 드러내고, 그동안 **위험 알림은 유지**됩니다. |
| 데이터가 조금 늦었는지 모른 채 조치한다 | **지연(stale) 채널**을 환자 단위로 드러냅니다. |
| 사고 후 “화면이 그랬는지” 말이 안 맞는다 | **감사 로그 + 동일 시나리오 재현(리플레이)**로 말을 맞출 수 있습니다. |

---

## 핵심 능력 (비개발자용)

- **운영 한 면:** 신뢰 상태, 교대 요약, 부하 지표, 큐, 최근 이벤트, 환자 상세가 **한 제품 톤**으로 묶여 있습니다.  
- **신뢰 UX:** “지금 믿어도 되는지”를 말로 덮지 않고, **모드·지연·동기화**를 화면에 남깁니다.  
- **운영 침착:** 과한 애니메이션·SF 대시보드 톤을 피하고, **긴 교대에 견디는 밀도**를 지향합니다.  
- **복구 가독성:** 재연결·제한 모드에서 **무엇이 잠기고, 언제 다시 조작 가능한지**가 같은 언어로 이어집니다.

---

## 운영자 입장에서의 흐름 (요약)

1. 상단에서 **신뢰도·복구 단계**를 확인합니다.  
2. **교대 요약**으로 열린 위급·대기·지연 채널 수를 훑습니다.  
3. **우선순위 큐**에서 행을 고르고, 오른쪽 **상세**로 맥락을 이어 읽습니다.  
4. **이벤트 흐름**으로 최근 변화의 맥을 짧게 추적합니다.  
5. (개발·검증 시에만) 하단 **검증·감사**를 열어 리플레이·불변식을 확인합니다.

---

## 스크린샷·시연 가이드

| 목적 | 권장 |
|------|------|
| **대표 메인 샷** | 브라우저 너비 **≥ 1100px**, 상단 **sticky**(제목·신뢰 strip)까지 포함한 **풀 높이** 캡처. |
| **가장 강한 workflow 순간** | 호흡 악화로 **위급이 큐 상단**에 올라온 직후(교대 요약에 열린 위급 ≥ 1). |
| **reconnect / degraded** | **제한 모드** 배너와 신뢰 strip이 **동시에** 보이는 프레임 한 장(“연결 흔들림에도 UI가 숨지 않음” 증명). |
| **stale** | 지연 환자 문구·행 배지가 같이 보이는 컷. |
| **피할 것** | Before/After 합성, **하단 `검증·감사 (개발)` 펼친 채**의 스크린샷, JSON·코드 덤프 위주 화면. |
| **톤** | 다크 기본; 대비는 **가독 우선**(과한 네온·장식 금지). |
| **발표 시 확대** | 큐 상단 2~3행 + 신뢰 strip만 잘라 **“위험 + 신뢰 경계”**를 크게. |

---

## 3–5분 데모 내러티브 (권장 순서)

1. **Opening:** “실시간 모니터링은 많은데, **교대·연결 끊김·데이터 지연**까지 한 화면에서 말해 주는 제품은 적다.”  
2. **악화:** 한 환자 호흡 악화가 **위급 큐**로 올라오는 것을 보여 준다.  
3. **인지·우선순위:** 큐와 상세가 **같은 맥락**으로 붙어 있음을 짚는다.  
4. **연결·제한:** 재연결·제한 모드에서 **조작이 잠기지만 위급은 남는** 이유를 짧게.  
5. **지연 데이터:** stale이 **“최신이 아님”**을 말하는 순간.  
6. **신뢰 복구:** 동기화·모드 문구가 **언제 다시 조작 가능한지**로 이어지는지.  
7. **Why it matters:** “운영자가 **잘못된 자신감**으로 누르지 않게 하는 것”이 목표.  
8. **Closing:** “기능 나열이 아니라, **운영 사고를 줄이는 화면**을 지향한다.”

기술·거버넌스·리플레이 깊이는 **말로 한 문장**만 하고, 궁금한 청중에게는 부록 링크로 넘깁니다.

---

## 포트폴리오 말할 거리 (Talking points)

- **제품 포인트:** 단일 운영 면 + 교대 요약 + 신뢰 strip — “패널 모음”이 아니라 **한 제품**.  
- **운영 포인트:** degraded에서도 **위급 가시성**을 포기하지 않는다는 메시지.  
- **UX 포인트:** 떨림 완화·좁은 단말 스택 — **실제 단말·교대**를 가정한 polish.  
- **엔지니어링 포인트(짧게):** 결정론 런타임·감사·리플레이 — “**사고 후 같은 화면을 다시 만들 수 있다**”는 한 줄로 압축.

## 데모 실행

```bash
cd dsd
npm install
npm run web:dev
```

브라우저에서 `http://localhost:3100` — 기본 페이지가 **시연 시나리오**(호흡 악화·재연결·제한 모드·지연)를 포함합니다.

```bash
npm test
npm run build -w @dsd/web
```
