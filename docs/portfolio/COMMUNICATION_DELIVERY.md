# DSD — Final Communication & Delivery Layer

**목적:** 면접·발표·포트폴리오에서 **상대가 빠르게 이해하고 기억**하도록 말하는 순서·한 줄·리스크를 고정한다.  
**금지:** 아키텍처 덤프, 리플레이/거버넌스 과설명, 코드 위주, “힘들게 만들었다” 서술, operator 가치 없는 기술 자랑.

---

## PHASE 1 — Executive Product Summary (30초)

| 항목 | 말할 내용 |
|------|-----------|
| **한 줄** | 중환·감시 구역에서 **누가 위험한지**, **데이터를 얼마나 믿을 수 있는지**, **연결이 흔들려도 화면이 조용히 실패하지 않는지**를 한 화면으로 묶은 **운영 콘솔 UI**다. |
| **푸는 운영 문제** | 알림만 쌓이고 **우선순위가 안 보이는 것**; 끊겼는데 **정상처럼 보이는 것**; 조금 늦은 데이터를 **최신인 줄 알고 누르는 것**; 사고 후 **“화면이 그랬다”** 말이 안 맞는 것. |
| **기존 모니터링과의 차이** | “보여 주기”를 넘어 **행동 순서(큐)**와 **신뢰 경계(모드·지연)**를 같은 화면에서 맞춘다. |
| **trust / degraded / stale** | 운영자가 **지금 믿어도 되는지**를 숨기지 않기 위함. 연결·동기 문제는 **제한 모드**로, 데이터 나이는 **지연 채널**로 드러낸다. |
| **의료 운영에서 왜 중요한가** | 잘못된 자신감 한 번이 **환자 안전·법적·교대 분쟁**으로 이어질 수 있어서, **말이 맞는 화면**이 필요하다. |

## PHASE 2 — Interview Narrative Architecture (문제 해결 흐름)

1. **왜 만들었는가** — 교대·실시간·불안정 연결 속에서도 **우선순위와 신뢰**가 무너지지 않게 하고 싶었다.  
2. **왜 ERP 맥락에서 command-center로 확장했는가** — ERP는 **흐름·기록·일관성**이 핵심인데, 중환은 그게 **초 단위·다 환자**로 압축된다. 같은 DNA로 **운영 한 벽**에 옮겼다.  
3. **프론트에서 무엇을 중요하게 봤는가** — **한 번에 읽히는 위계**, **끊겼을 때의 말투**, **클릭이 환자와 말이 통하는지**.  
4. **determinism을 왜 봤는가** — “내 화면”과 “로그·사후 재현”이 **같은 이야기**여야 교대·감사에서 싸움이 줄어든다.  
5. **operational calmness** — 긴 근무에서 **깜빡임·소음**이 곧 오판이다.  
6. **가장 어려웠던 점** — truth와 화면 순서가 어긋날 때 **조용히 멀어지지 않게** 하는 것.  
7. **어떻게 해결했는가** — 단일 셸·안정화 규칙·제한 모드 잠금·짧은 운영 문구로 **한 제품 톤**으로 묶었다.  
8. **무엇을 배웠는가** — 강한 런타임만으로는 부족하고, **전달 가능한 운영 언어**가 같이 가야 납품처럼 보인다.

## PHASE 3 — Technical Depth Compression (한 줄씩)

| 주제 | 한 줄 |
|------|--------|
| **Replay verification** | 같은 이벤트 줄을 다시 돌리면 **큐·상태가 항상 같게** 나와야 사고 후에도 말이 맞는다. |
| **Append-only audit** | 지운 척하지 않고 **쌓인 순서가 곧 진실**이다. |
| **Degraded UX** | 나쁠 때 **버튼을 잠그되 위급은 숨기지 않는다**. |
| **Queue stabilization** | 들쭉날쭉한 순서 변화를 **한 박자 줄여** 스캔할 수 있게 한다. |
| **Operational trust** | “지금 이 화면을 믿어도 되는가”를 **한 줄·한 배지**로 맞춘다. |
| **CI / release governance** | 중요한 불변식은 **머지할 때부터** 깨지면 안 된다고 자동으로 막는다. |

## PHASE 4 — Demo Delivery Strategy

| 단계 | 할 일 |
|------|--------|
| **첫 5초** | 전체 화면 한 번 — “**한 제품**”이라고만 짚는다. |
| **다음 20초** | 신뢰 strip + 교대 요약 — **지금 믿을 수 있는지·열린 위급**만. |
| **본론** | 큐 상단 2행 + 상세 — **누가 위험한지**만 말한다. |
| **멈춤 포인트** | “여기서 조작이 잠깁니다” — **제한 모드** 배너에서 한 번 멈춘다. |
| **reconnect / degraded** | 위 멈춤과 **같은 프레임**에서 연결·동기 이야기를 한다. |
| **stale** | 지연 문구 한 줄 — **최신이 아닐 수 있음**만. |
| **생략** | KPI 숫자 해설, 이벤트 타입 나열, **하단 검증 패널**은 열지 않는다. |
| **시간** | **3분 목표**, 질문 많으면 5분까지. |
| **질문 유도** | “사고 나고 화면이랑 로그랑 다르면 어떻게 하시나요?” → 리플레이 한 줄로 연결. |

## PHASE 6 — Portfolio Talking Points (짧게 강하게)

- **Engineering:** 같은 입력이면 **같은 큐** — 사고 후에도 말이 맞는다.  
- **Frontend:** 한 화면·한 톤·**끊김에도** 위급은 남긴다.  
- **UX:** 침착한 밀도 — **긴 교대**를 가정했다.  
- **Operational:** 제한 모드에서 **조용한 실패**를 금지한다.  
- **Product:** 모니터링이 아니라 **운영 결정**을 돕는 면이다.  
- **Realism:** 병상·이름·지연 문구를 **데모 수준에서도** 현장에 가깝게 잡았다.
